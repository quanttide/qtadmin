# 量潮咨询 PRD

> 量潮科技旗下四条业务线之一，属于"报时"侧的业务经营主体。

## 业务特征

咨询业务与其他三条业务线的核心区别：

| 维度 | 量潮咨询 | 数据/课堂/云 |
|------|---------|-------------|
| 交付形态 | 方案+报告 | 软件/课程/平台 |
| 周期 | 1-6个月 | 数周至数月 |
| 复购驱动 | 信任+口碑 | 产品体验 |
| 产能瓶颈 | 顾问人力 | 基础设施 |
| 知识沉淀 | 行业认知+方法论 | 代码/课程/配置 |

咨询的产能不来自工具，而来自人。系统能为咨询团队做的事不是"自动化"，而是增强顾问的判断力和客户管理能力。

## 场景一：这个客户值不值得投入

**决策频率**：每月数次。**决策者**：咨询业务负责人。

### 决策现场

客户档案卡片展示：

1. **客户画像** — 行业、规模、数字化成熟度评估（L0-L4）
2. **接触历史** — 历次沟通纪要、需求变化轨迹、决策链成员变动
3. **项目匹配度** — 当前团队能力能否覆盖客户需求，绿/黄/红三档
4. **战略价值** — 该客户在行业内的标杆效应，是否有助于建立行业认知

负责人点"接"后，系统自动：创建项目空间 → 分配顾问 → 锁定行业知识库。

### 替代路径

积累足够多的客户决策后，系统在客户接触初期即提示："这家客户与之前某项目的画像相似度 82%，最终结果为成功交付，建议接洽。"

## 场景二：方案怎么对被客户接受

**决策频率**：方案推进期间每1-2周。**决策者**：项目顾问。

### 决策现场

方案推进看板显示：

1. **客户反馈状态** — 每个方案模块的接受度（已认可/讨论中/有异议/未接触）
2. **异议焦点** — 客户有异议的模块，记录具体顾虑和团队回应策略
3. **决策链状态** — 关键决策人的态度变化轨迹，谁支持、谁中立、谁反对

顾问不需要记"客户上次怎么说"——打开看板就知道上次沟通后客户的态度变化，以及谁还没表态。

### 替代路径

系统识别出异议模式："连续三个项目客户都对数据安全方案有异议"，触发知识库更新：在后续项目的方案模板中，将数据安全部分的论证前置，附上行业案例和合规依据。

## 场景三：项目结束后留下了什么

**决策频率**：每个项目结束后。**决策者**：咨询业务负责人。

### 决策现场

系统在项目关闭时自动生成沉淀清单：

1. **行业认知** — 该项目产出的行业洞察、客户业务理解、竞争格局分析
2. **方法论资产** — 方案框架、评估模型、交付流程中被验证有效的环节
3. **复用价值评估** — 该资产可以复用到哪些行业/场景，预计节省多少启动成本

负责人确认后，资产自动归入对应行业知识库，标记成熟度（草稿/已验证/已标准化）。

### 替代路径

新项目启动时，系统自动关联相关行业的知识资产："该客户属于制造业，知识库中有 3 个制造业项目的沉淀，建议在方案阶段优先参考。"

## 场景四：顾问能力怎么在项目中成长

**决策频率**：持续。**决策者**：咨询业务负责人 + 顾问本人。

### 决策现场

顾问的个人看板上：

1. **当前项目贡献** — 在项目中的角色、负责模块、产出物列表
2. **能力成长轨迹** — 每个项目结束后评估的技能增长（行业理解/方案设计/客户沟通）
3. **待提升领域** — 系统根据项目中的反馈和成果评估，提示可重点提升的方向

顾问本人看到的是成长路线，负责人看到的是团队能力热力图——哪个行业方向的人手充足，哪个方向存在缺口。

### 替代路径

系统识别出某顾问连续两个项目在客户沟通环节被评价为"待提升"，自动推送沟通技巧课程和案例库；同时在下个项目分配时，为该顾问搭配沟通能力强的搭档。
