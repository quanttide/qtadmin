# p01: 两院制议事机制

验证公司议事体系的可执行性。

## 设计意图

创始人视角的核心目的：**在控制与规模之间找平衡。**

| 矛盾 | 解决方案 |
|------|----------|
| 事必躬亲 → 成为瓶颈 | 章程授权两院处理日常事项 |
| 完全放手 → 失去控制 | 保留剩余权力 + 终审权 |
| 独裁决策 → 缺乏合法性 | 两院审议引入集体智慧和代表性 |
| 分权制衡 → 陷入僵局 | 创始人特权可打破僵局 |

用让渡换取成长空间，用保留确保不失控。让渡日常治理权使公司不被创始人的精力天花板所限，保留特权确保公司在关键时刻不偏离创始人的核心意图。这是一个可进化的框架——随着两院成熟，创始人可以逐步让渡更多。

## 政体模型：君主立宪制（早期）

创始人是"君主"，公司实行君主立宪制。借鉴英国立宪早期模式：

- 创始人是大股东和实际控制人，两院的权力由创始人让渡
- 上议院（股东代表大会）权力压倒下议院
- 章程已授权的事项由上议院或下议院按程序处理
- 章程**未授权**的事项，默认由创始人直接行使（君主保留特权）
- 一票否决权是创始人特权的一个子项，而非全部

### 两院运行模式

两院的运行分为三种模式：

#### 模式一：独立行使

各院在其专属职权范围内独立决策，不经另一院流转。

| 机构 | 独立行使的权力 | 未来方向 |
|------|--------------|----------|
| 上议院（股东代表大会） | 司法权（裁决章程争议） | 未来将司法权交给董事会 |
| 下议院（公司代表大会） | 部分立法权 | — |

#### 模式二：两院立法流程

需要两院协作的事项按以下方向流转：

| 流向 | 适用范围 | 流程 |
|------|----------|------|
| **下议院 → 上议院** | 标准事项 | 下议院发起 → 下议院审议/表决 → 上议院审议/表决 → 创始人确认 |
| **上议院 → 下议院** | 少数重大事项 | 上议院发起 → 上议院审议/表决 → 下议院审议/表决 → 创始人确认 |

标准事项从下议院发起，经上议院审议后由创始人确认。少数重大事项（如章程修改）从上议院发起，经下议院知晓或审议后由创始人确认。

#### 模式三：创始人直接行使

章程未授权事项，由创始人独立决策，无两院流程。

### 创始人特权行使路径

创始人直接行使权来自于**君主保留原则**——章程未授权的事项，默认为创始人直接行使。创始人也可选择将事项提交两院按立法流程处理，获取集体决策意见后再确认。

| 路径 | 流程 | 性质 |
|------|------|------|
| **通过两院立法流程** | 按两院流程审议/表决 → 创始人确认 | 创始人**选择**引入审议程序，获取集体决策意见 |
| **直接行使** | 创始人判定 → 创始人行使 → 记录 | 章程未授权事项的默认路径，创始人是独立决策者 |

两条路径的核心区别：前者**有审议环节**，创始人是最终确认者；后者**无审议环节**，创始人是独立决策者。两者均须形成书面记录，但审议记录有无是区分关键。

### 与一票否决权的关系

一票否决权是创始人多项特权之一。根据场景不同，可选择通过上议院行使或直接行使。

## 体系特征

1. **三模共存** — 独立行使、两院流转、创始人直接行使三种模式同时存在，由事项性质而非层级决定走哪条路
2. **权力非对称** — 上议院压倒下议院，但下议院在部分立法权上独立行使，不是简单的上下级关系
3. **进化预留** — 上议院的司法权标注了"未来→董事会"，架构本身预留了权力转移路径，不是静态的
4. **创始人弹性边界** — 创始人不是高高在上的第三层，而是在两院流程中做终审确认、在两院独立行使时不介入、在章程未授权时才直接行使
5. **授权密度驱动** — 章程授权越细，两院独立行使和立法流程的覆盖面越大；章程未授权越多，创始人直接行使的空间越大。这是一个动态治理

## 场景

### 下议院（公司代表大会）
- 代表选举：各业务域/职能域按席位分配规则选举代表
- 议案审议：代表提出议案，经审议后进入表决
- 表决机制：简单多数 / 三分之二多数
- 罢免程序：对不履职代表的罢免流程

### 上议院（股东代表大会）
- 立宪权：制定和修改章程的提案 → 审议 → 表决
- 司法权：裁决章程争议的受理 → 审理 → 裁决
- 创始人部分特权通过上议院行使
- 议事记录归档

### 创始人特权
- **章程未授权事项**：直接行使，视为君主保留权力
- **一票否决权**：合规风险 / 战略偏离 / 两院僵局触发行使
- **直接发布权**：在章程未明确禁止的范围内直接发布指令
- **其他保留特权**：根据章程规定保留的其他未让渡权力
- 行使流程：触发 → 判定 → 行使 → 记录
- 所有特权行使均不得委托他人

## 验收标准

- [ ] 政体模型（君主立宪制）定位清晰，权力来源明确
- [ ] 两院职权划分清晰，上议院权力压倒下议院
- [ ] 下议院选举、审议、表决流程完整可执行
- [ ] 上议院立宪与司法流程完整
- [ ] 创始人特权大于一票否决权，包含多项子特权
- [ ] 每项特权的行使路径明确（通过上议院 vs 直接行使）
- [ ] 通过上议院行使时有完整的审议记录（上议院审议 + 表决 + 创始人否决）
- [ ] 直接行使时无审议环节，创始人是独立决策者
- [ ] 两种路径均包含：触发 → 判定 → 行使 → 记录
- [ ] 审议记录的有无是区分两种路径的核心标志
- [ ] 章程未授权事项默认为创始人直接行使
- [ ] 存在不得委托他人的限制机制
- [ ] 所有特权行使形成书面记录，包含时间、事由、依据、结果
